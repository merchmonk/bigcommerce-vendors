# Change Safety Note

## Destructive Changes Policy

Never delete files, folders, or data unless all of the following are true:

1. A complete replacement source has been created and verified.
2. A recovery path exists (git commit/branch backup or exported copy).
3. A before/after inventory has been checked for parity.
4. Explicit user confirmation has been given for the deletion.

If any item is uncertain, do not delete.
